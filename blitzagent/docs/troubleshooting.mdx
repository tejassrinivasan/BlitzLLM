This guide helps you diagnose and resolve common issues when using the Blitz API.

## API Connection Issues

<AccordionGroup>
  <Accordion icon="wifi" title="Connection Errors">
    **Problem**: Unable to connect to the Blitz API
    
    **Solutions**:
    1. **Check API endpoint**: Ensure you're using `https://blitzagent.onrender.com`
    2. **Verify API key**: Confirm your Blitz API key is correct and active
    3. **Check network**: Ensure your network allows HTTPS requests
    
    **Test connection**:
    ```bash
    curl -X GET "https://blitzagent.onrender.com/health" \
      -H "Authorization: Bearer YOUR_BLITZ_API_KEY"
    ```
  </Accordion>
  
  <Accordion icon="key" title="Authentication Errors (401)">
    **Problem**: Receiving 401 Unauthorized responses
    
    **Solutions**:
    1. **Verify API key format**: Use `Authorization: Bearer YOUR_BLITZ_API_KEY`
    2. **Check for extra spaces**: Ensure no leading/trailing spaces in your key
    3. **Confirm key is active**: Contact support if key appears correct
    
    **Example request**:
    ```bash
    curl -X POST "https://blitzagent.onrender.com/analyze" \
      -H "Authorization: Bearer YOUR_BLITZ_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"query": "Test query"}'
    ```
  </Accordion>
  
  <Accordion icon="clock" title="Rate Limiting (429)">
    **Problem**: Too Many Requests error
    
    **Solutions**:
    1. **Slow down requests**: Wait before retrying
    2. **Check rate limits**: Review your usage patterns
    3. **Implement backoff**: Use exponential backoff for retries
    
    **Rate limit info**:
    - 100 requests per minute per API key
    - 10 concurrent requests maximum
  </Accordion>
</AccordionGroup>

## Query Issues

<AccordionGroup>
  <Accordion icon="search" title="No Results or Poor Responses">
    **Problem**: Queries returning empty or poor quality responses
    
    **Solutions**:
    1. **Be more specific**: Include player names, team names, or specific metrics
    2. **Add context**: Use the `extra_context` field for additional details
    3. **Check spelling**: Ensure correct spelling of player/team names
    
    **Good query examples**:
    ```json
    {
      "query": "LeBron James points per game this season",
      "extra_context": "Include shooting percentages and recent performance"
    }
    ```
  </Accordion>
  
  <Accordion icon="exclamation-triangle" title="Error Messages">
    **Problem**: Receiving error messages in responses
    
    **Common errors and solutions**:
    - **"Invalid query format"**: Ensure JSON is properly formatted
    - **"Rate limit exceeded"**: Wait before making more requests
    - **"Service unavailable"**: Temporary issue, try again later
  </Accordion>
</AccordionGroup>

## Integration Issues

<AccordionGroup>
  <Accordion icon="code" title="API Integration Problems">
    **Problem**: Issues integrating Blitz into your application
    
    **Solutions**:
    1. **Check content type**: Use `Content-Type: application/json`
    2. **Verify request format**: Ensure proper JSON structure
    3. **Handle responses**: Check for both success and error responses
    
    **Example integration**:
    ```python
    import httpx
    
    async def query_blitz(query_text):
        headers = {
            "Authorization": "Bearer YOUR_BLITZ_API_KEY",
            "Content-Type": "application/json"
        }
        
        try:
            async with httpx.AsyncClient() as client:
                response = await client.post(
                    "https://blitzagent.onrender.com/analyze",
                    headers=headers,
                    json={"query": query_text}
                )
                response.raise_for_status()
                return response.json()
        except httpx.HTTPError as e:
            print(f"Request failed: {e}")
            return None
    ```
  </Accordion>
  
  <Accordion icon="stream" title="Streaming Issues">
    **Problem**: Issues with the streaming endpoint
    
    **Solutions**:
    1. **Use Server-Sent Events**: The `/analyze/stream` endpoint uses SSE
    2. **Handle streaming**: Use appropriate libraries for SSE handling
    3. **Test with stream viewer**: Use the built-in stream viewer tool
    
    **Stream viewer**: Navigate to `https://blitzagent.onrender.com/stream_viewer.html`
  </Accordion>
</AccordionGroup>

## Performance Issues

<AccordionGroup>
  <Accordion icon="zap" title="Slow Response Times">
    **Problem**: API responses are taking too long
    
    **Solutions**:
    1. **Check query complexity**: Simpler queries respond faster
    2. **Use appropriate endpoint**: Use `/analyze` for standard queries
    3. **Monitor service status**: Check if there are any service issues
    
    **Optimization tips**:
    - Keep queries focused and specific
    - Avoid overly complex multi-part questions
    - Use streaming for complex analysis
  </Accordion>
  
  <Accordion icon="refresh" title="Timeout Issues">
    **Problem**: Requests timing out
    
    **Solutions**:
    1. **Increase timeout**: Set longer timeout values in your client
    2. **Use streaming**: Switch to `/analyze/stream` for long queries
    3. **Break up queries**: Split complex questions into simpler parts
  </Accordion>
</AccordionGroup>

## Status Monitoring

You can check the current status of the Blitz API:

- **Health endpoint**: `https://blitzagent.onrender.com/health`
- **Service status**: Monitor response times and availability
- **Rate limit headers**: Check remaining quota in API responses

If you're experiencing ongoing issues, please contact our support team with:
- Your API key (first/last 4 characters only)
- Example queries that are failing
- Error messages you're receiving
- Timestamp of when issues occurred
